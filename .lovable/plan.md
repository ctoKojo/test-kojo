# Fix Routing Conflict ‚Äî Pathless `_authenticated` Layout

## Problem

TanStack Router throws:

```
Conflicting configuration paths for route "/"
Files: src/routes/_authenticated/route.tsx AND src/routes/index.tsx
```

Cause: `_authenticated/route.tsx` declared `createFileRoute("/_authenticated")` but lives in a folder. With a folder-based layout, the route already represents the layout ‚Äî combined with `index.tsx` it produces a duplicate `/` registration after the gen step. We want a **pathless layout** (no URL segment) that wraps protected pages, while keeping clean URLs like `/dashboard`, `/students`, etc.

## Target route structure

```text
src/routes/
‚îú‚îÄ‚îÄ __root.tsx                 shell only (unchanged)
‚îú‚îÄ‚îÄ index.tsx                  smart redirect: / ‚Üí /dashboard or /login
‚îú‚îÄ‚îÄ login.tsx                  public login page (unchanged stub)
‚îî‚îÄ‚îÄ _authenticated/
    ‚îú‚îÄ‚îÄ route.tsx              pathless auth guard layout (Outlet)
    ‚îî‚îÄ‚îÄ dashboard.tsx          protected dashboard at /dashboard
```

Pathless = folder name starts with `_`, so `_authenticated` contributes **no URL segment**. Children render at their own clean paths (`/dashboard`, later `/students`, `/sessions`, `/payments`, ...).

## Changes

### 1. `src/routes/_authenticated/route.tsx`
Keep as the pathless layout. The path string in `createFileRoute` stays `"/_authenticated"` (this is the route ID, not a URL segment ‚Äî TanStack strips the leading `_` segment from the URL). Component renders `<Outlet />`. Auth guard wiring (`beforeLoad` redirect to `/login`) stays a TODO comment for Phase 0 Chunk 1 ‚Äî no logic added in this fix.

### 2. `src/routes/index.tsx`
Replace placeholder with a redirect-only route:

```tsx
import { createFileRoute, redirect } from "@tanstack/react-router";

export const Route = createFileRoute("/")({
  beforeLoad: () => {
    // TODO Chunk 1: check supabase session and redirect to /dashboard if signed in
    throw redirect({ to: "/login" });
  },
});
```

### 3. `src/routes/_authenticated/dashboard.tsx` (new)
Protected dashboard placeholder at `/dashboard`. Minimal stub now; real content lands in later phases.

```tsx
import { createFileRoute } from "@tanstack/react-router";

export const Route = createFileRoute("/_authenticated/dashboard")({
  component: () => (
    <div className="p-8">
      <h1 className="text-2xl font-semibold">Dashboard</h1>
      <p className="text-sm text-muted-foreground">
        Protected route ‚Äî Phase 0 Chunk 1 will wire role-based content.
      </p>
    </div>
  ),
});
```

### 4. `src/routeTree.gen.ts`
Auto-regenerated by the TanStack Router Vite plugin on the next dev/build cycle. We do not hand-edit it; the previous manual entry will be overwritten correctly once the conflict is gone.

## Verification

1. Dev server starts with no "Conflicting configuration paths" error.
2. Visiting `/` redirects to `/login`.
3. Visiting `/login` shows the existing login stub.
4. Visiting `/dashboard` renders the new dashboard stub (auth guard still TODO).
5. `src/routeTree.gen.ts` lists `/`, `/login`, `/_authenticated`, `/_authenticated/dashboard`, with `/_authenticated` as a pathless layout (no URL segment in `fullPath`).
